**Method:** Assess → Plan → Implement → Verify → Refine  
**Source of truth:** SEO BIBLE 2026 uploaded July 12, 2026  
**Hard rule:** Part 10 primary-source corrections override conflicting secondary guidance.

The executable authority is `scripts/validate_product_proof_program.py`. It verifies twenty independent concerns:

1. Requirement identity
2. Source traceability
3. Status vocabulary
4. Implemented artifacts
5. Implemented tests
6. External blockers
7. Claim registry
8. Deprecation registry
9. Primary-source pack
10. Bounded crawler
11. Rule engine
12. Root-cause reporting
13. Artifact manifest
14. Agent contribution ledger
15. CLI product path
16. Command overlay
17. Capability overlay
18. Skill convergence
19. Feedback learning and optimization loops
20. Release truth

A pass means the implementation and traceability contracts exist and are internally consistent. It does not mean an authorized live website was successfully audited, a clean external installation was reproduced, or the public release gate was approved.
