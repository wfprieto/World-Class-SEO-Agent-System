<a id="experimentation"></a>
## Search experimentation

Define hypothesis, unit, exposure, guardrails, duration, interference risks, and stopping rules before interpreting results. Before-after movement does not establish causation.

<a id="reporting-forecasting"></a>
## Reporting and forecasting

Preserve source dates, scope, uncertainty, seasonality, denominator changes, and model assumptions. Forecasts are scenarios, not guaranteed outcomes.

<a id="api-limitations"></a>
## API limitations

Record quotas, pagination, sampling, thresholding, missing rows, anonymization, and provider-specific semantics. Missing permission or configuration must not become empty success.

<a id="cms-frameworks"></a>
## CMS and framework patterns

Diagnose generated HTTP and rendered output rather than assuming platform behavior. Defaults, plugins, themes, and deployment layers can change SEO-critical output.

<a id="drift-observability"></a>
## Drift and observability

Use compatible baselines, integrity checks, schema versions, bounded retention, and explicit severity rules. Missing history is insufficient evidence, not no change.
